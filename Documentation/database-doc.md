Database Documentation: upazila_text.db1. OverviewThe upazila_text.db is an SQLite 3 database designed to serve as the foundational text corpus for a Retrieval-Augmented Generation (RAG) system. It contains highly informative, structured textual data about administrative sub-districts (Upazilas) in Bangladesh.The data was extracted from a WordPress XML export (WXR), specifically targeting a custom post type (upazila), and mapped from Advanced Custom Fields (ACF) into a flat, relational schema. The database has been strictly pruned of all empty or placeholder data to ensure high-quality vector embeddings.2. Schema ArchitectureThe database consists of two tables that share an identical schema.Tablesupazilas: The primary operational table containing the cleaned, production-ready dataset (40 records).informative_upazilas: A secondary/legacy table mirroring the schema. Currently unused in the main pipeline but maintained for structural backward compatibility.Table Definition (SQL)SQLCREATE TABLE upazilas (
    wp_id INTEGER PRIMARY KEY,
    title TEXT NOT NULL,
    development_info TEXT,
    travel_info TEXT,
    geographic_info TEXT,
    history_info TEXT
);

Source Mapping (The ACF Integration)Because the original data source is a WordPress site utilizing Advanced Custom Fields (ACF), the XML <wp:meta_key> tags do not directly match the SQLite column names.The data pipeline (generate_upazila_db.py) translates the ACF fields into the database schema using the following exact mapping:XML development-info ➔ SQLite development_infoXML travel-guide-data ➔ SQLite travel_infoXML geo-location-data ➔ SQLite geographic_infoXML upazila-history-data ➔ SQLite history_infoNote: Any WordPress items outside the upazila custom post type or lacking a publish status were excluded during extraction.5. Data Quality & Current StateThe database has undergone a strict sanitization process (prune_db.py) using a frequency-based (plateau) algorithm to remove "noise" that would degrade vector search performance.Total Records: 40Removed Data: 455 records were purged because they contained identical 61-character default placeholder strings (এই উপজেলার উন্নয়ন ও অগ্রযাত্রার তথ্য শীঘ্রই আপডেট করা হচ্ছে।).Compression: The database file has been optimized via the VACUUM command to reclaim disk space from the deleted rows, resulting in a lightweight, high-density file.6. Recommended RAG Extraction QueryWhen querying this database to chunk and vectorize the text for your LLM, it is recommended to concatenate the fields into a single rich document string per upazila to maintain semantic context:SQLSELECT 
    wp_id,
    title,
    'History: ' || IFNULL(history_info, '') || 
    ' Geography: ' || IFNULL(geographic_info, '') || 
    ' Travel: ' || IFNULL(travel_info, '') || 
    ' Development: ' || IFNULL(development_info, '') AS full_context
FROM upazilas;
